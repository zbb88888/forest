## 系统依赖关系

**详细文档**：`docs/game/system/dependency.md`

**依赖原则**：
1. 依赖关系必须是单向的（避免循环依赖）
2. 基础系统优先实现
3. 高层系统依赖低层系统
4. 同层系统之间尽量减少依赖

**系统分层**：
```
第0层：核心基础层
└── 游戏状态系统

第1层：环境基础层
├── 地图系统
└── 时间系统

第2层：环境增强层
└── 光照系统

第3层：实体基础层
└── 玩家系统

第4层：资源系统层
└── 资源系统

第5层：内容系统层
├── 植物系统
└── 机器人系统

第6层：增强系统层
└── 装备系统

第7层：界面系统层
└── UI系统
```

**开发优先级**：从第0层开始，逐层向上实现。详细依赖关系和优先级请参考 `dependency.md`。

## 文档维护

### 更新原则
1. 每个系统有独立的详细文档
2. 系统有重大更新时更新对应文档
3. 新增系统时创建新文档并更新索引
4. 保持文档与代码同步

